🔧 Setup Instructions

Quick Start (XSS Demo Only)

The XSS demo works immediately without any setup. Simply click "Launch Demo" above!

Full Setup (All Demos)

To run all demos with full functionality, you'll need:

1. Install Node.js

Download and install Node.js from https://nodejs.org/

2. Install MySQL (Optional)

For SQL injection demo functionality, install MySQL or MariaDB.

3. Run the Server

After installing Node.js, run these commands:

# Install dependencies
npm install

# Start the server
node server.js

# Or use the PowerShell script (Windows)
.\start-server.ps1

4. Access the Demos

Once the server is running, visit:

  • XSS Demo: http://localhost:3000/
  • Health Check: http://localhost:3000/health
  • Security Info: http://localhost:3000/api/security-info

🤖 ChatGPT Learning Guide for Students

Maximize your learning experience with AI-powered assistance

🎯 How to Use ChatGPT Effectively for Web Security Learning

💡

Ask Specific Questions

Instead of "What is XSS?", ask "Can you explain how reflected XSS works with a practical example and show me how to prevent it?"

🔍

Request Code Analysis

Share vulnerable code snippets and ask: "Can you identify the security vulnerabilities in this code and suggest fixes?"

🛠️

Get Step-by-Step Explanations

Ask for detailed breakdowns: "Walk me through the process of a SQL injection attack step by step"

📝 Sample Prompts for Each Demo

🔍 XSS Demo Prompts

Understanding: "Explain the difference between stored, reflected, and DOM-based XSS with examples"
Code Review: "Review this JavaScript code for XSS vulnerabilities: [paste your code]"
Prevention: "Show me 5 different ways to prevent XSS attacks in modern web applications"

💉 SQL Injection Prompts

Attack Vectors: "What are the most common SQL injection attack vectors and how do they work?"
Secure Coding: "Convert this vulnerable SQL query to use parameterized statements: [paste query]"
Testing: "How can I test my application for SQL injection vulnerabilities?"

🔄 CSRF Protection Prompts

Concept: "Explain CSRF attacks with a real-world scenario and show how tokens prevent them"
Implementation: "Show me how to implement CSRF protection in Node.js/Express applications"

🎓 Learning Strategies

📚

Progressive Learning

Start with basic concepts, then ask for advanced scenarios and edge cases

🔬

Hands-on Practice

Ask ChatGPT to create practice exercises and vulnerable code for you to fix

🎯

Real-world Context

Request examples from actual applications and industry best practices

🔄

Iterative Learning

Follow up with "Can you explain this part in more detail?" or "What if...?" questions

⚠️ Important Guidelines

✅ Always verify ChatGPT's suggestions with official documentation
✅ Test security implementations in safe, controlled environments
✅ Use ChatGPT to understand concepts, not as the sole source of truth
❌ Never use AI-generated code in production without thorough review
❌ Don't share sensitive or real production code with AI tools
